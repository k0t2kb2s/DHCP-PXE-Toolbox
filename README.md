# PXE DHCP Whitelist Server

## Описание проекта

Автономный PXE-сервер (DHCP + TFTP) с веб-панелью управления и MAC-вайтлистингом для сетевой установки ОС Debian Linux.

Сервер принимает DHCP Discover от устройств в локальном физическом сегменте сети, показывает обнаруженные MAC-адреса в веб-панели, выдает сетевые настройки только разрешенным клиентам и передает им PXE-загрузчик через TFTP.

## Структура папки tftpboot

В корне папки `tftpboot` должны лежать файлы PXE-загрузки Debian:

```text
tftpboot/
├── pxelinux.0
├── linux
├── initrd.gz
└── pxelinux.cfg/
    └── default
```

Назначение файлов:

- `pxelinux.0` - PXE-загрузчик, который получает целевой ПК по TFTP.
- `linux` - ядро установщика Debian Linux.
- `initrd.gz` - initrd-образ установщика Debian Linux.
- `pxelinux.cfg/default` - конфигурация меню PXE-загрузки.

## Развертывание на реальном железе

### 1. Физическое подключение

Соедините серверный ноутбук и целевой ПК сетевым кабелем напрямую или подключите оба устройства к одному физическому свитчу.

На целевом ПК включите загрузку по сети в BIOS/UEFI и выберите PXE Boot для проводного сетевого интерфейса.

### 2. Настройка статического IP на сервере

Назначьте статический IPv4-адрес сетевому интерфейсу серверного ноутбука. Пример:

```text
IP-адрес: 192.168.100.1
Маска:    255.255.255.0
```

Пример для Linux через `ip`:

```bash
sudo ip addr flush dev enp3s0
sudo ip addr add 192.168.100.1/24 dev enp3s0
sudo ip link set enp3s0 up
```

Замените `enp3s0` на имя вашего сетевого интерфейса.

### 3. Конфигурация пула в config.json

Настройте DHCP-пул в `config.json` под выбранную подсеть:

```json
{
  "dhcp_settings": {
    "pool_start": "192.168.100.50",
    "pool_end": "192.168.100.150",
    "subnet_mask": "255.255.255.0",
    "router": "192.168.100.1",
    "pxe_next_server": "192.168.100.1",
    "pxe_boot_file": "pxelinux.0"
  },
  "whitelist": []
}
```

`pxe_next_server` должен совпадать с IP-адресом серверного ноутбука.

### 4. Запуск приложения через sudo

DHCP-серверу нужен доступ к сетевому интерфейсу и raw-пакетам, поэтому запускайте приложение через `sudo`:

```bash
sudo DHCP_ENABLED=true DHCP_INTERFACE=enp3s0 uvicorn app:app --host 0.0.0.0 --port 8000
```

Замените `enp3s0` на интерфейс, к которому подключен целевой ПК или свитч.

## Использование системы

Откройте веб-панель управления на серверном ноутбуке:

```text
http://localhost:8000
```

Чтобы добавить новое устройство:

1. Подключите целевой ПК к серверному ноутбуку напрямую или через свитч.
2. Включите целевой ПК и запустите PXE-загрузку.
3. Дождитесь появления MAC-адреса нового устройства в веб-панели.
4. Добавьте найденный MAC-адрес в вайтлист.
5. Включите тумблер PXE для этого MAC-адреса.
6. Перезапустите PXE-загрузку на целевом ПК, если он уже получил отказ до добавления в вайтлист.

После включения PXE для разрешенного MAC-адреса сервер выдаст клиенту IP-адрес из пула, адрес TFTP-сервера и загрузочный файл `pxelinux.0`.

## Важное примечание: траблшутинг

Перед стартом сервера отключите системный файрвол на хосте, иначе он может блокировать исходящие бродкаст UDP-пакеты DHCP и TFTP-трафик на порт 69.

Для `firewalld`:

```bash
sudo systemctl stop firewalld
```

Для правил `iptables`:

```bash
sudo iptables -F
```

Если DHCP-запросы видны в панели, но клиент не получает загрузчик, проверьте:

- целевой ПК и сервер находятся в одном физическом сетевом сегменте;
- на серверном интерфейсе задан статический IP из той же подсети, что и DHCP-пул;
- `pxe_next_server` в `config.json` равен IP-адресу серверного ноутбука;
- в `tftpboot` есть `pxelinux.0`, `linux`, `initrd.gz` и `pxelinux.cfg/default`;
- TFTP не блокируется файрволом, особенно UDP-порт 69.
